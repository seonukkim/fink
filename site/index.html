How it works

FInk reads the document on the device…

  1. 1Input — photo · image · PDF · pasted clause
  2. 2OCR & page reconstruction (Korean + English)
  3. 3Clause segmentation
  4. 4Evidence retrieval from official sources
  5. 5Financial risk signals + cash-flow scenarios
  6. 6Review report (the four outputs below)

Four outputs

Shown side by side, never merged into one total-loss number.

Review priority

Which clauses deserve a closer look, in order.

Money: low / base / high

A cost range from values or editable assumptions.

Time

When the money moves: timing, recoupment, term, delays.

Confidence

How sure the reading is; unclear data lowers confidence.
